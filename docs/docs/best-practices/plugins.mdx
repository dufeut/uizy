# Plugin Development

## Use Descriptive Namespaces

Choose clear, short namespace names that describe the plugin's purpose:

```javascript
// ✅ Good namespaces
app.plugin("ui", { ... });     // UI components
app.plugin("fx", { ... });     // Effects and animations
app.plugin("auth", { ... });   // Authentication
app.plugin("form", { ... });   // Form handling

// ❌ Avoid vague or overly long names
app.plugin("myPlugin", { ... });
app.plugin("utilityHelpers", { ... });
```

## Keep Plugins Focused

Each plugin should solve one problem well:

```javascript
// ✅ Focused plugin
function TooltipPlugin(app) {
  app.plugin("tooltip", {
    directives: {
      show: (el, { value, modifiers }) => { /* tooltip logic */ },
      hide: (el) => { /* hide logic */ },
    },
    components: {
      dark: () => "bg-gray-900 text-white px-2 py-1 rounded text-sm",
      light: () => "bg-white shadow px-2 py-1 rounded text-sm",
    },
  });
}

// ❌ Plugin doing too many unrelated things
function KitchenSinkPlugin(app) {
  app.plugin("stuff", {
    directives: { tooltip: ..., ripple: ..., validate: ... },
    actions: { login: ..., fetchData: ..., saveFile: ... },
    // Too many unrelated features
  });
}
```

## Provide Sensible Defaults

Always handle missing options gracefully:

```javascript
function ThemePlugin(app, options = {}) {
  const {
    defaultTheme = "light",
    transitionDuration = 300,
    storageKey = "app-theme",
  } = options;

  // Plugin logic using defaults...
}
```
